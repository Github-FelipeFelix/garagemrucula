# Aprendizados herdados (Rebobina 3D + EnergyDex)

Fontes: `Rebobina 3D/rebobina3d` (Next 16 + Supabase + Vercel + PWA, maduro) e `AppEnergy/energydex` (Next 16 + React 19 + Tailwind v4 + Supabase SSR + PWA).

1. **PostgREST: nunca embedded joins** (`.select('*, x:y(*)')`) — sem FK constraints retorna `data:null` silencioso. Sempre 2 queries + join manual no TS.
2. **Admin protegido por e-mail hardcoded no `src/middleware.ts`** (`ADMIN_EMAILS`). Deslogado→login (?next); logado sem permissão→home; API→401. Rate-limit opcional (Upstash) em rotas sensíveis.
3. **Migrations = arquivos `.sql`** em `supabase/migrations/`, rodados no SQL Editor do Supabase (DDL não passa pela REST API). Código degrada com segurança até a coluna existir.
4. **Serverless: sempre `await`** efeitos (upload/e-mail). Fire-and-forget é abandonado quando a função responde.
5. **PWA:** manifest único (`start_url`, `scope:"/"`, `display:standalone`, ícones 192/512/maskable) + **service worker NETWORK-FIRST para HTML** (deploy atualiza na hora), cache-first só em assets com hash. Nunca cache-first em HTML.
6. **Supabase clients (SSR):** `lib/supabase/client.ts` (`createBrowserClient`, anon key) e `server.ts` (`createServerClient` com cookies do `next/headers`). Service role só no servidor p/ escrita sensível.
7. **RLS desde o dia 1:** leitura pública do que é público (carros); escrita só admin.
8. **Storage de imagens:** Supabase Storage; `next.config.ts` com `images.remotePatterns` p/ `**.supabase.co`.
9. **Deploy:** git push → Vercel. **git user.email deve bater com a conta Vercel** (`felipeherrera.contato@gmail.com`), senão bloqueia deploy no Hobby. TS error quebra o build.
10. **Testar de verdade:** tsc/build ≠ funciona. Validar na UI com Chrome REAL `--remote-debugging-port=9222` + `dev-browser --connect` (Google bloqueia login em browser de automação), checar console, desktop E mobile, após deploy. `window.confirm` nativo trava no dev-browser → dá pra sobrescrever.
11. **Design tokens** em `globals.css` (CSS vars) + Tailwind v4. Texto sobre fundo escuro usa VARIANTE CLARA da cor da marca.
12. **dev-browser:** roda script JS em sandbox QuickJS (sem npm/fs host). `browser.getPage`, `page.goto/click/fill/evaluate/screenshot`. **Timeout de 30s por script** → dividir QA em scripts curtos. `setInputFiles` com path do host FALHA (fs bloqueado) → testar upload por outra via (ex: signed URL via REST). Daemon pode ficar órfão.

## Aprendizados novos — sessão Garagem Rúcula (01–02/07/2026)
13. **npm no Google Drive CORROMPE** (EBADF/EPERM; o `G:` nem aceita junction/reparse). Projetos Next ficam em disco LOCAL (`C:\dev\...`), sincronizados via GitHub — não pela pasta do Drive.
14. **`git clone` NÃO herda o `user.email` local** → o commit sai com o e-mail GLOBAL (ex: `felipe.felix@housi.com.br`). Rodar `git config user.email felipeherrera.contato@gmail.com` em CADA clone, senão a Vercel bloqueia o deploy.
15. **Vercel Hobby + repo PRIVADO bloqueia deploy** se o commit author/co-author não for membro ("commit author did not have contributing access"). Solução definitiva: **repo PÚBLICO** (some a trava); alternativa: author = dono + sem co-authors.
16. **SW cache-first em `/_next/static` → FALSO "hydration mismatch" no DEV** (serve bundle JS velho enquanto o server tem o novo). Em PROD NÃO afeta (assets têm hash de conteúdo). Ao investigar hydration no dev-browser: desregistrar o SW + `caches.delete()` antes de testar.
17. **Registro.br: apex/raiz = campo Nome VAZIO** (não `@`, que Cloudflare et al. usam). `A`→IP no raiz (vazio); `CNAME`→`www`.
18. **NUNCA versionar senha/segredo** — o repo é PÚBLICO. `.env.local` no `.gitignore`; senha do admin fica fora do git.
19. **supabase-js v2 exige Node 22 (WebSocket nativo) em script standalone** — dentro do Next funciona (Next provê WebSocket). Pra scripts Node avulsos (seed/admin/QA), usar a **REST API do Supabase via `fetch`**.
20. **lucide-react removeu ícones de marca** (Instagram etc) → criar SVG próprio, COM `width`/`height` explícitos (senão o SVG estica sem constraint).
21. **Next 16: renomear `middleware.ts` → `proxy.ts`** (função `middleware`→`proxy`; `config.matcher` igual).

## Aprendizados novos — sessão 2 Garagem Rúcula (01/07/2026, tarde/noite)
22. **`NEXT_PUBLIC_*` é embutida no BUILD** → se `NEXT_PUBLIC_SITE_URL` na Vercel tiver `localhost` (copiado do `.env.local`), sitemap/robots/canonical/og/QR saem com localhost em PROD. Fazer o helper `siteUrl()` DEFENSIVO: env não-local vence; senão, em deploy Vercel (`process.env.VERCEL`) usa o domínio canônico; no client, QR/share usam `window.location.origin`. Nunca copiar `.env.local` inteiro pra Vercel sem revisar.
23. **Card clicável com carrossel interno:** NÃO aninhar `<button>` dentro de `<a>` (HTML inválido → hydration mismatch). Padrão: o card é um `<div>`; foto e texto são `<Link>` IRMÃOS pro mesmo destino; setas/bolinhas são `<button>` irmãos com z-index acima + `preventDefault`/`stopPropagation`; swipe não dispara click (o browser distingue), então trocar foto ≠ navegar.
24. **next/image (Next 16):** `sizes` baixo (ex: `120px`) + quality default (75) = foto BORRADA. Pra qualidade: declarar `images.qualities` no `next.config` (senão `quality={90}` é ignorado) + `formats:['image/avif','image/webp']`. O arquivo ORIGINAL vai intacto pro Storage — a degradação é só na exibição.
25. **Google OAuth via Supabase:** (a) Google Cloud → OAuth Client "Web" com redirect `https://<ref>.supabase.co/auth/v1/callback` (o dono do projeto Cloud pode ser QUALQUER conta — não aparece no login); (b) Supabase → Auth → Providers → Google ON + Client ID/Secret + Site/Redirect URLs. Código: `signInWithOAuth({provider:'google', redirectTo:.../auth/callback})` + rota `/auth/callback` com `exchangeCodeForSession` (PKCE). Erro `"provider is not enabled"` = não ligou/salvou no Supabase. App "Testing" = só test users + aviso "não verificado"; **publicar** (scopes email/profile = não-sensíveis) NÃO exige verificação. Google **bloqueia login em browser automatizado** → validar só até a tela do Google; login completo é manual.
26. **`cursor:pointer` não é default em `<button>`/`<select>`** (só `<a>` tem) → parecem "não clicáveis". Regra global no `globals.css` pra `button`/`select`/`[role=button]`/`summary` + `not-allowed` nos disabled.
27. **Falso "hydration mismatch" no DEV por hot-reload** (além do SW, item 16): após várias edições, server e client podem ficar com versões diferentes do componente (ex: className antigo no diff do warning). NÃO é bug real → confirmar no **build de produção** (`npm run build && npm start` → hydration 0). Se persistir no dev, limpar `.next` com o dev parado.

## Aprendizados novos — sessão 3 (02/07/2026, PC 2)
28. **Alvo de toque mobile ≥32px + ação explícita em vez de drag:** ícone 12px + `p-1` ≈ 20px é INTOCÁVEL no dedo (mínimo recomendado ~44px; 32px já resolve em tile pequeno). E ação essencial NUNCA pode existir só via drag: no admin, "trocar capa" era só arrastar-pra-1ª-posição pela alça (dnd-kit com listeners SÓ na alça → tocar na foto não faz nada) — pro leigo no celular = "não funciona". Fix: botão explícito ("⭐ capa" → `arrayMove(i→0)`); drag vira atalho secundário.
29. **dev-browser daemon órfão v2 (named pipe) + bypass Playwright:** daemon quebra com `EADDRINUSE \\.\pipe\dev-browser-daemon-<user>`; se o processo velho for de sessão elevada, `taskkill /F` dá "Acesso negado" (sem UAC não mata). Limpar `daemon.pid`/`daemon-spawn.lock` não basta enquanto o pipe estiver preso. **Bypass que funciona:** usar o Playwright que o dev-browser instala em `~/.dev-browser/node_modules` DIRETO num script Node (`require('C:/Users/<user>/.dev-browser/node_modules/playwright')`; chromium já está em `%LOCALAPPDATA%\ms-playwright`). Bônus: `newContext({ hasTouch:true, isMobile:true, viewport:{width:390,...} })` + `locator().tap()` = QA com TOQUE real, mais fiel que click pra bug mobile.

## Aprendizados novos — sessão 4 (02/07/2026, acesso admin no celular + segurança)
30. **`next start` local acusa 404 de `/_vercel/insights/script.js` — FALSO alarme:** o Vercel Analytics injeta esse `<script>`, mas o arquivo só é servido pela infra da Vercel em PRODUÇÃO; em localhost dá SEMPRE 404, 100% inofensivo. Não perder tempo investigando — confirmar com `curl -o /dev/null -w "%{http_code}"` em prod (200). Junto com #16 (SW) e #27 (hot-reload), é mais um "erro só no dev/local" que não é bug.
31. **Proteção do `/admin` NÃO depende de esconder a URL** — pode ter link público (ex: rodapé) sem enfraquecer, porque a trava real é em camadas: whitelist de e-mail (`lib/admins.ts`) → `proxy.ts` (`getUser()` revalida a sessão no servidor Auth, não só o cookie) → cada rota `/api/admin/*` re-checa `getAdminUser()` (defesa em profundidade) → RLS no banco (o anon key é PÚBLICO por design e vai no bundle; só lê o que é público [`cars`], insere `leads`, executa `bump_car_view` — NÃO lê `leads`/`car_sales` nem escreve carro; a escrita é service role, `import "server-only"`, nunca no cliente). Auditei as 4 nesta sessão: sólidas. O elo mais fraco então é a **conta de e-mail** → recomendar **2FA na conta Google** do admin.
32. **PWA `shortcuts` no manifest + "adicionar à tela inicial"** = acesso rápido a uma rota interna (ex: `/admin`) no celular sem digitar URL. `MetadataRoute.Manifest` do Next aceita `shortcuts:[{name, short_name, description, url, icons}]` (conferir no `.d.ts` antes — TS error quebra o build da Vercel). O ícone do shortcut precisa existir (apontei pro `/icon-192.png`, que é 200). Complemento sem código: no celular, abrir a rota → menu → "Adicionar à tela inicial" cria um atalho que abre direto nela (Android Chrome e iPhone Safari).

## Aprendizados novos — sessão 5 (02/07/2026, galeria / mobile / pré-testes)
33. **Grid que só define colunas em `sm:`/`lg:` ESTOURA o layout viewport no CELULAR (bug do "super zoom"):** um `<div class="grid ... lg:grid-cols-2">` SEM `grid-cols-1` na base vira `display:grid` com coluna implícita `auto`, que dimensiona pelo `max-content` do conteúdo (texto/imagem). No mobile isso pode passar de 1400px → o navegador do celular EXPANDE o layout viewport (`window.innerWidth` foi de 390 pra **1504**) e a página abre "com zoom"/cortada. A meta viewport estava CERTA (`width=device-width`); o culpado é o grid. Fix: sempre `grid-cols-1` na base (`minmax(0,1fr)` deixa a coluna encolher). **CRÍTICO pra QA:** Playwright/dev-browser com `viewport` simples (SEM `isMobile`) TRAVA o layout viewport na largura pedida e ESCONDE essa classe de bug — só pega com `...devices["iPhone 13"]` (tem `isMobile:true`, HONRA a meta viewport como celular real). Medir `window.innerWidth` vs `visualViewport.width` denuncia o estouro.
34. **Troca de foto INSTANTÂNEA = pré-carregar a vizinha na MESMA URL do next/image:** o delay ao clicar "próxima" (medido ~1,8s em 4G) é a imagem baixando na hora. Fix confiável: renderizar uma janela `[anterior, atual, próxima]` EMPILHADA no mesmo box (mesmo `sizes`/`quality` → MESMA URL otimizada → cache hit garantido), atual `opacity-100`, vizinhas `opacity-0` + `loading="eager"`. Nos cards da vitrine/home, disparar o pré-carregamento só no 1º `pointerenter`/`touchstart` (senão pesa o load inicial de N cards). Verificar: a `<img>` da vizinha já está `complete && naturalWidth>0` ANTES do clique.
35. **Tailwind v4: `border` (bare) numa pílula colorida vira "bordinha preta":** o Preflight do v4 usa `border-color: currentColor` por padrão → `border` + `text-black` = borda preta, mesmo com `border-transparent` no mix (conforme ordem/precedência das utilities). Em selos/badges sólidos, NÃO usar `border` na base; pôr borda só onde quer de fato (ex.: selo "vendido" com `border border-white/30`). E selos que aparecem juntos precisam das MESMAS classes de tamanho (padding/fonte) senão ficam de tamanhos diferentes.
36. **`isolate` (isolation:isolate) contém o z-index dos filhos:** um selo `absolute z-30` DENTRO de um card vaza pro contexto de empilhamento da PÁGINA se o card não for um contexto próprio (`relative` sem z-index/transform NÃO cria contexto). Aí empata com o painel de filtros `sticky z-30` e, por vir depois no DOM, pinta POR CIMA ao rolar. Fix limpo: `isolate` no card → o z-30 do selo fica preso lá dentro; o card cai pro nível 0 no root e o filtro (z-30) fica acima. Melhor que guerra global de z-index.
37. **Dev local usa o Supabase REAL (compartilhado com produção):** apagar/editar carro no `/admin` reflete NA HORA no localhost (carro apagado no meio do teste → página vira 404, e o `notFound()` está CERTO). Consequências: (a) abrir páginas de carro em testes INCREMENTA o contador de views real (`ViewTracker`/`bump_car_view`); (b) `getCarBySlug`/`getCars` retornam `null`/`[]` em QUALQUER erro (transitório incluso) → 404/vitrine vazia indistinguível de "não existe" (pendência: retry curto). Em QA, só rodar o fluxo público read-only (anon key) — nunca escrever no banco.

## Aprendizados novos — sessão 6 (02/07/2026, revisão geral / fase de testes)
38. **Satori (next/og) NÃO aplica EXIF orientation** — foto de celular (retrato gravado como paisagem + EXIF) sai DEITADA na OG image, mesmo aparecendo certa no site (next/image aplica EXIF). Fix elegante: passar a capa pelo otimizador do próprio site (`/_next/image?url=...&w=1200&q=75`) — corrige rotação e, como o fetch do Satori não manda `Accept: image/webp`, devolve JPEG (formato que o Satori lê). O `w` precisa existir nos deviceSizes e o `q` nas `qualities` do next.config.
39. **Duplicar registro que REFERENCIA arquivos de storage = bomba armada:** se o duplicate copia só o array de paths (não os objetos) e o delete remove os objetos do bucket, apagar original OU cópia mata as mídias do sobrevivente. Padrão correto: duplicate COPIA os objetos (`storage.from(b).copy(src, dest)` — na REST: `POST /object/copy {bucketId, sourceKey, destinationKey}`) e o delete só remove paths que NENHUMA outra row referencia (query nas rows + Set de paths em uso). Bônus: o PATCH deve limpar do bucket as mídias removidas na edição (senão lixo eterno), com a MESMA checagem de compartilhamento.
40. **Decodificar percent-encoding em bash (`printf '%b' ${var//%/\\x}`) é traiçoeiro** — produz URL malformada silenciosamente e o curl devolve código `000` (nem conectou). Um teste meu acusou "56 fotos quebradas" que estavam 100% OK. Código `000` = URL/rede inválida, NÃO recurso quebrado. Pra verificar mídia: Node com `decodeURIComponent`/fetch, ou melhor ainda direto na fonte (REST do Supabase com anon key lendo `photos[].url` — sem contaminação dos cards de "veja também" que aparecem no HTML da página).
41. **Regra nova `react-hooks/set-state-in-effect` (ESLint React 19) acusa padrões CORRETOS** (fechar menu em mudança de pathname, ler querystring pós-mount, esconder banner em standalone). Next 16 não roda ESLint no build → não bloqueia deploy. Tratar como dívida cosmética; NÃO refatorar às pressas código testado só pra calar o lint.

## Aprendizados novos — sessão 7 (02/07/2026, feedback do PWA + autonomia do admin)
42. **`scrollIntoView(block:"nearest")` rola a PÁGINA, não só o container** — pra centralizar um filho numa tira `overflow-x-auto`, o scrollIntoView sobe/desce a página inteira pra trazer o elemento (no celular = "a tela fica descendo" ao trocar de foto). Solução: rolar SÓ o container — `container.scrollBy({ left: thumb.getBoundingClientRect().left - strip.getBoundingClientRect().left - (strip.clientWidth - thumb.clientWidth)/2, behavior:"smooth" })`. Bônus: `overflow-x-auto` força `overflow-y:auto` → clipa o `ring`/`border` do filho no topo; resolver com padding vertical na tira (`py-1.5`).
43. **Login Supabase no mobile/PWA:** (a) OAuth — o redirect automático do `signInWithOAuth` às vezes NÃO dispara no navegador do celular / PWA standalone (botão fica em loading e nada acontece). Fix: `options:{ skipBrowserRedirect:true }` + `window.location.href = data.url` (redirect manual). (b) e-mail/senha — depois do `signInWithPassword`, `router.replace()+router.refresh()` (nav client) pode ficar "carregando" eterno no mobile porque o servidor demora a enxergar o cookie novo; usar `window.location.assign(next)` (navegação HARD → o servidor lê o cookie na hora). QA com conta real: entra em ~1,5s.
44. **CMS-lite: conteúdo do site editável pelo admin sem quebrar nada.** Tabela `site_settings` (1 linha, coluna `data` jsonb; RLS: anon SELECT, escrita só service_role). `getSiteSettings()` com React `cache()` + merge sobre DEFAULTS → se a tabela não existir/vier vazia, cai nos textos fixos (o site NUNCA quebra — degrada, regra 3). Escrita numa rota `/api/admin/*` que faz `revalidatePath` das páginas afetadas (muda na hora). As páginas públicas viram ISR (`export const revalidate`). Escopo p/ usuário LEIGO: só texto de marketing; NÃO expor o que quebra o negócio (WhatsApp/links/preço). A migração DDL o dono roda 1x no SQL Editor — o **MCP do Supabase pode estar logado noutra conta** (aqui estava): `execute_sql`/`apply_migration` retornam "You do not have permission" pro projeto que não é da conta logada.

## Aprendizados novos — sessão 8 (13/07/2026, área de Peças + galeria do espaço)
45. **Feature nova SEPARADA sem arriscar produção — o padrão que deu certo aqui:** (a) **tabela própria** (`parts`) espelhando a existente, com libs próprias (`part-input`/`parts-queries`/`parts-storage`) — NÃO tocar nos arquivos críticos do fluxo antigo (`storage-media.ts`/`car-input.ts`/`queries.ts`/`api/admin/cars`); (b) **bucket compartilhado com PREFIXOS** (`car-media` + `cars/`/`parts/`/`espaco/`) evita criar bucket/policy novos, e cada helper de cleanup consulta **só a sua tabela** e remove **só os seus paths** → o cross-delete do #39 fica impossível ENTRE áreas (paths são UUID, prefixos distintos); (c) **parametrizar componentes compartilhados por prop com default do fluxo antigo** (`PhotoUploader folder="cars"`, `ViewTracker carId|partId`, `QrCard base="carros"`) mantém o existente byte-a-byte e o tsc pega qualquer caller esquecido; (d) tudo **degrada pra vazio** sem a migração (leitura com try/catch → `[]`, seções somem). Resultado: 12 arquivos novos + 19 aditivos, carros intocados.
46. **Migrar registro entre tabelas SEM perder mídia (mover carro→peça):** COPIAR os arquivos pra novos paths na tabela destino (a nova linha vira dona), apagar a linha origem e remover do bucket **só os paths órfãos que a nova linha NÃO adotou** (`oldPaths.filter(p => !adoptedByNew.has(p))`). Se a cópia cair no fallback (mesmo path), ele fica de fora da remoção → a mídia da nova linha nunca some. Provado no banco/Storage real (foto da peça lê 200 após apagar o carro).
47. **Verificar remoção no Storage do Supabase: o public URL tem CACHE de CDN** — logo após um DELETE (que retorna 200), o `GET .../object/public/...` ainda pode devolver 200 por minutos (edge cache). Pra confirmar de verdade que sumiu, usar a **API de listagem** (`POST /storage/v1/object/list/<bucket>` com `prefix`/`search`) — é a fonte da verdade, sem CDN. (Junto com #40, mais uma pegadinha de QA de storage.)
48. **CMS-lite jsonb aceita campos novos sem migração:** adicionar `aboutPhotos` ao `site_settings.data` (jsonb) não precisou de DDL — só estender o tipo + `mergeSettings`/sanitize + o form. Migração só é necessária pra COLUNA/TABELA nova, não pra chave nova dentro de um jsonb existente.
